## 📐 ПРИНЦИПЫ ПЛАНИРОВАНИЯ

### Принцип 1: Атомарность задачи
Одна задача = одно логическое изменение = один коммит.

```
❌ ПЛОХО: "Сделай систему авторизации"
✅ ХОРОШО:
  Задача 1: Создать модель User в БД (DATABASE)
  Задача 2: Создать Pydantic схемы UserCreate, UserResponse (DEVELOPER)
  Задача 3: Написать сервис регистрации auth_service.register() (DEVELOPER)
  Задача 4: Написать endpoint POST /api/auth/register (DEVELOPER)
  Задача 5: Написать тесты на регистрацию (TESTER)
  Задача 6: Написать сервис и endpoint логина (DEVELOPER)
  Задача 7: Реализовать JWT в httpOnly cookie (DEVELOPER)
  Задача 8: Написать тесты на логин и JWT (TESTER)
```

### Принцип 2: Правильный порядок (зависимости)
Задачи выполняются в порядке зависимостей:

```
БД (DATABASE) → Модели → Схемы → Сервисы → Роутеры → Тесты → Ревью → Деплой
```

### Принцип 3: Указывать агента для каждой задачи
Каждая задача помечена кому передать:

```
[DATABASE]   — задачи по схеме БД и миграциям
[DEVELOPER]  — задачи по написанию кода
[TESTER]     — задачи по написанию тестов
[REVIEWER]   — финальная проверка блока задач
[DEVOPS]     — задачи по деплою и инфраструктуре
[DOCUMENTER] — задачи по документации
```

### Принцип 4: Размер задачи
- Задача должна выполняться за одну сессию (не больше 2-3 часов работы)
- Если задача больше — разбей ещё мельче
- Правило: если задачу нельзя описать одним предложением — она слишком большая

## 🔍 САМОАУДИТ ПЕРЕД СОХРАНЕНИЕМ TASKS.md

Перед тем как сохранить файл — проверь каждую задачу на три вопроса:

**1. Что если данные пустые или некорректные?**
Пример: пользователь отправил пустой email, 0 вместо числа.
→ Если нет задачи для TESTER на валидацию — добавь.

**2. Что если два пользователя делают это одновременно?**
Пример: двое регистрируются с одним email в одну секунду.
→ Если нет задачи на уникальность/атомарность — добавь.

**3. Что если токен авторизации истёк в момент запроса?**
Пример: пользователь долго не активен, JWT протух.
→ Если нет задачи на обработку 401 — добавь.

Нашёл пропуск → добавил задачу → только потом сохраняй.
