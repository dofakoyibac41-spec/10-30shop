## 🔴 OWASP TOP 10 — ПОЛНЫЙ ЧЕКЛИСТ

### A01 — Broken Access Control (Нарушение контроля доступа)

```
[ ] Каждый protected endpoint проверяет JWT токен?
[ ] Admin endpoints недоступны обычным пользователям?
[ ] Пользователь не может получить данные другого пользователя?
[ ] Нет прямого обращения к объектам по ID без проверки владельца?
      (Пример: GET /api/users/123 — убедись что user_id из токена == 123)
[ ] Нет "скрытых" endpoints без авторизации?
[ ] Права проверяются на каждом уровне (route + service)?
```

### A02 — Cryptographic Failures (Слабая криптография)

```
[ ] Пароли хэшируются через bcrypt (cost factor 12+)?
[ ] Нет MD5, SHA1, SHA256 для паролей?
[ ] JWT подписан алгоритмом HS256 или RS256 (не none)?
[ ] Секретный ключ JWT минимум 32 символа случайной строки?
[ ] HTTPS везде (нет HTTP в продакшн)?
[ ] SSL сертификат не истёк?
[ ] Чувствительные данные не хранятся в открытом виде в БД?
[ ] Логи не содержат паролей, токенов, API ключей?
```

### A03 — Injection (Инъекции)

```
[ ] Нет SQL конкатенации — только SQLAlchemy ORM?
[ ] Нет f"SELECT ... {user_input}" нигде в коде?
[ ] Входные данные валидируются Pydantic на все endpoints?
[ ] Нет eval() или exec() с пользовательскими данными?
[ ] Нет shell=True в subprocess с пользовательскими данными?
[ ] Нет XSS — пользовательский контент не рендерится как HTML?
```

### A04 — Insecure Design (Небезопасная архитектура)

```
[ ] Rate limiting на auth endpoints (register, login)?
[ ] Нет возможности перебора (brute force) паролей?
[ ] Нет возможности перебора лицензионных ключей?
[ ] Email enumeration защита (одинаковый ответ при неверном email и пароле)?
[ ] Лимиты на размер запросов (нет возможности залить 1GB файл)?
[ ] Токены имеют срок жизни (не вечные)?
```

### A05 — Security Misconfiguration (Неверная конфигурация)

```
[ ] CORS только для разрешённых доменов (не *)?
[ ] Debug режим выключен в продакшн?
[ ] Stack trace не возвращается пользователю в ошибках?
[ ] Swagger /docs закрыт в продакшн (или защищён паролем)?
[ ] .env файл не доступен через веб (Nginx блокирует)?
[ ] Стандартные пароли БД изменены?
[ ] Docker не запускает контейнеры от root?
[ ] Nginx заголовки безопасности настроены?
```

### A06 — Vulnerable Components (Уязвимые зависимости)

```
[ ] Python зависимости актуальны (pip audit)?
[ ] Node.js зависимости актуальны (npm audit)?
[ ] Нет известных CVE в используемых версиях?
[ ] Docker образы обновлены до последних patch версий?
```

```bash
# Команды аудита зависимостей
cd backend && pip install pip-audit && pip-audit   # Python
cd frontend && npm audit                            # Node.js
npm audit --audit-level=high                       # только высокие/критические
```

### A07 — Authentication Failures (Проблемы аутентификации)

```
[ ] JWT хранится в httpOnly cookie (не localStorage)?
[ ] Refresh токен ротируется при каждом использовании?
[ ] Logout инвалидирует refresh токен в БД?
[ ] Нет возможности использовать старый токен после logout?
[ ] При смене пароля — все сессии инвалидируются?
[ ] Нет фиксации сессии (session fixation)?
```

### A08 — Integrity Failures (Нарушение целостности)

```
[ ] Данные от пользователя не используются напрямую как команды?
[ ] Нет десериализации ненадёжных данных?
[ ] Webhook запросы проверяют подпись (если есть)?
```

### A09 — Logging Failures (Проблемы логирования)

```
[ ] Все попытки авторизации логируются (успех и неудача)?
[ ] Логи не содержат чувствительных данных?
[ ] Логи доступны для просмотра (не теряются)?
[ ] Подозрительная активность (много неудачных логинов) видна в логах?
```

### A10 — SSRF (Server-Side Request Forgery)

```
[ ] Если принимаешь URL от пользователя — валидируешь его?
[ ] Нет возможности через URL обратиться к внутренним сервисам?
[ ] Whitelist разрешённых доменов для внешних запросов?
```

## 💼 БИЗНЕС-ЛОГИКА БЕЗОПАСНОСТИ (SaaS-специфика)

Эти уязвимости не входят в стандартный OWASP но критичны для SaaS продукта:

### Mass Assignment — подмена привилегированных полей

```
[ ] Пользователь НЕ может обновить поле plan через API?
      Тест: PATCH /api/users/me с {"plan": "max"} → должен вернуть 422 или игнор
[ ] Пользователь НЕ может обновить поле is_active, role, is_admin?
[ ] Pydantic схемы Update НЕ содержат привилегированных полей?
```

### Горизонтальное повышение привилегий (IDOR)

```
[ ] GET /api/users/{id} — проверяется что id == текущий пользователь?
[ ] GET /api/orders/{id} — проверяется что заказ принадлежит пользователю?
[ ] Нет перебора ID (UUID вместо integer — защита от sequential scan)?
```

### Обход тарифного плана

```
[ ] Pro/Max endpoints проверяют план через middleware, не только в UI?
[ ] Rate limits применяются по тарифу (free=10 req/min, pro=100 req/min)?
[ ] При истечении подписки доступ к Pro функциям закрывается немедленно?
```

### Утечка секретов в Git

```bash
# Проверить что секреты не закоммичены
git log --all --full-history -- "*.env" | head    # .env файлы в истории
git grep -i "api_key\|secret\|password" -- "*.py" "*.ts"  # хардкод в коде
git log -p | grep -i "sk-\|api_key\|secret" | head -20   # в diff истории

# Проверить .gitignore покрывает всё нужное
cat .gitignore | grep -E "\.env|secret|key"
```

## 🌐 ПРОВЕРКА NGINX ЗАГОЛОВКОВ

```nginx
# Эти заголовки ОБЯЗАНЫ быть в конфиге Nginx:
add_header X-Frame-Options "DENY";
add_header X-Content-Type-Options "nosniff";
add_header X-XSS-Protection "1; mode=block";
add_header Referrer-Policy "strict-origin-when-cross-origin";
add_header Content-Security-Policy "default-src 'self'";
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains";
```

Проверить можно на: https://securityheaders.com

## 🔑 ПРОВЕРКА JWT

```
[ ] Алгоритм: HS256 или RS256 (никогда не "none")
[ ] Payload не содержит пароль или чувствительные данные
[ ] Access token: срок 15 минут
[ ] Refresh token: срок 30 дней, хранится в БД
[ ] При logout refresh token удаляется из БД
[ ] Подпись проверяется на каждом запросе
```

---

## 🐳 ПРОВЕРКА DOCKER БЕЗОПАСНОСТИ

```
[ ] Контейнеры не запускаются от root (USER node / USER python)
[ ] Секреты через переменные окружения, не в Dockerfile
[ ] .dockerignore исключает .env, .git, node_modules
[ ] Образы используют конкретные теги (не :latest)
[ ] Внутренние сервисы (db) не открыты наружу (нет 5432:5432)
[ ] Только nginx открыт на 80/443
```
