## БАГ-1 — Блоки слишком высокие (Hero и Категории)

| | |
|---|---|
| **Приоритет** | 🟡 Средний |
| **Страница** | Главная `/` |
| **Компонент** | `HomePage.vue` |

**Описание:**
Hero-секция («Одевайся как ты живёшь») и секция с сеткой категорий не вмещаются в экран ПК по высоте — требуют скролла чтобы увидеть весь блок. Каждый блок выглядит чрезмерно растянутым вниз.

**Ожидаемое поведение:**
Каждый блок (Hero, Категории) должен помещаться в пределах видимой области экрана (viewport) на стандартном разрешении ПК (1920×1080, 1440×900).

**Как исправить:**
Уменьшить `min-height` у секций Hero и Categories в `HomePage.vue` и/или `tokens.css`. Не убирать полностью, просто скорректировать вниз (например, с `100vh` до `80vh` или фиксированного значения).

## БАГ-2 — Серый фон в незаполненных ячейках сетки

| | |
|---|---|
| **Приоритет** | 🟡 Средний |
| **Страница** | Главная `/`, Каталог `/catalog` |
| **Компонент** | `HomePage.vue`, `CatalogPage.vue` |

**Описание:**
Когда категорий меньше чем ячеек в CSS-сетке (например, 5 категорий в сетке на 8 мест), пустые ячейки отображаются как серые прямоугольники. Аналогично в каталоге при малом количестве товаров. Выглядит как незаконченная страница с плейсхолдерами.

**Скриншот:** прилагается (см. скриншот с серым блоком под категорией «Тест»)

**Ожидаемое поведение:**
Сетка должна адаптироваться под реальное количество элементов. Пустых серых ячеек быть не должно.

**Как исправить:**
Убрать фиксированное количество колонок через `grid-template-columns` с явным числом ячеек. Использовать `auto-fill` или `auto-fit` вместе с `minmax()` чтобы сетка заполнялась только реальным контентом:
```css
grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
```

## БАГ-4 — Кнопка «О нас» в навигации ведёт к секции категорий

| | |
|---|---|
| **Приоритет** | 🔴 Высокий |
| **Страница** | Все страницы (навигация) |
| **Компонент** | `NavBar.vue`, `HomePage.vue` |

**Описание:**
При клике на «О нас» в навигационной шапке страница скроллится к секции с категориями, а не к блоку о бренде. Навигация работает некорректно.

**Ожидаемое поведение:**
Клик на «О нас» → плавный скролл к секции «О бренде» на главной странице.

**Как исправить:**
Проверить атрибут `id` на блоке «О бренде» в `HomePage.vue` — он должен совпадать с якорем в `NavBar.vue`. Например:
```html
<!-- HomePage.vue — блок о бренде -->
<section id="about">...</section>

<!-- NavBar.vue -->
<RouterLink to="/#about">О нас</RouterLink>
```
Убедиться, что `id="about"` стоит именно на секции о бренде, а не на секции категорий.

## БАГ-5 — Ошибка JSON при удалении категории или товара в админке

| | |
|---|---|
| **Приоритет** | 🔴 Критический |
| **Страница** | Админка `/admin` |
| **Компонент** | `useApi.js` → функция `apiFetch` |

**Описание:**
При нажатии кнопки «Удалить» у категории или товара появляется ошибка:
```
JSON.parse: unexpected end of data at line 1 column 1 of the JSON data
```
Удаление не выполняется (или выполняется, но UI показывает ошибку).

**Скриншот:** прилагается

**Причина:**
`DELETE /api/categories/:id` и `DELETE /api/products/:id` возвращают статус `204 No Content` с пустым телом ответа. Функция `apiFetch` в `useApi.js` безусловно вызывает `res.json()` на любом ответе — при пустом теле это вызывает ошибку парсинга JSON.

**Как исправить:**
В функции `apiFetch` добавить проверку статуса перед парсингом:
```js
// useApi.js — в функции apiFetch
if (res.status === 204 || res.headers.get('content-length') === '0') {
  return null; // пустой ответ — не парсить
}
return res.json();
```

|---|---|

| **Страница** | Все страницы (шапка) |
| **Компонент** | `NavBar.vue` |

**Описание:**
Текущий логотип в виде картинки выглядит некорректно в шапке — плохо вписывается в дизайн.

**Ожидаемое поведение:**
Логотип должен выглядеть чисто и читаемо.

**Как исправить:**
Заменить `<img>` логотипа на текстовый вариант — «10:30 AM» в фирменном шрифте дизайн-системы:
```html
<!-- NavBar.vue -->
<RouterLink to="/" class="nav__logo">10:30 AM</RouterLink>
```
```css
.nav__logo {
  font-family: var(--font-display);
  font-size: var(--fs-label-sm);
  letter-spacing: var(--ls-label);
  text-transform: uppercase;
  color: var(--color-primary);
  text-decoration: none;
}
```
